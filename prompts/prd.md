# Product Requirements Document (PRD) Prompt

Use this prompt to generate a structured PRD for a new feature or product.

---

## Context

> Describe the product or feature and the business problem it addresses.

## Sections to Generate

1. **Overview** — What is being built and why?
2. **Goals & Non-Goals** — What this PRD covers and what it does not.
3. **User Stories** — Key user scenarios in `As a [user], I want [action], so that [benefit]` format.
4. **Functional Requirements** — Detailed feature requirements.
5. **Non-Functional Requirements** — Performance, security, accessibility, scalability.
6. **Dependencies** — External systems, APIs, or teams involved.
7. **Open Questions** — Unresolved decisions that need stakeholder input.

## Output Format

Generate a structured Markdown document following the sections above.
Use a priority label (P0 / P1 / P2) for each requirement.
